The Challenge

Agricultural mushroom production requires yield estimation from existing camera feeds, but manual counting is time-consuming and error-prone.

Technical Challenges

  • Dense clustering: Mushrooms grow close together, often touching
  • Variable lighting: Growing environments have inconsistent illumination
  • No training data: Limited labeled examples for ML approaches
  • Time sensitivity: Client needed solution in 6-8 weeks
  • Hardware constraints: Must run on standard computers

Why Classical CV Over ML?

  • ✓ No training data required: Rules-based approach
  • ✓ Faster delivery: 6 weeks vs 12+ weeks for data collection + training
  • ✓ Deterministic: Predictable behavior, easier debugging
  • ✓ Lower cost: No GPU requirements or ML infrastructure
  • ✓ Transparent: Client can understand and tune parameters

Multi-Stage Processing

Pipeline Stages

1. Preprocessing
   ├─ CLAHE (Contrast Limited Adaptive Histogram Equalization)
   ├─ Gaussian blur (noise reduction)
   └─ Adaptive thresholding

2. Segmentation
   ├─ Binary masking (mushroom vs background)
   ├─ Morphological operations (opening/closing)
   └─ Connected components analysis

3. Separation (Key Innovation)
   ├─ Distance transform on binary mask
   ├─ Watershed algorithm for clustered mushrooms
   └─ Contour detection and filtering

4. Measurement
   ├─ Dual-radius estimation:
   │  ├─ Method 1: Bounding box dimensions
   │  └─ Method 2: Contour area → radius
   └─ Average for robust sizing

Key Algorithm: Distance Transform + Watershed

# Distance transform for separating clustered mushrooms
import cv2
import numpy as np
from scipy import ndimage

# Apply distance transform
dist_transform = cv2.distanceTransform(binary_mask, cv2.DIST_L2, 5)

# Find peaks (mushroom centers)
local_max = ndimage.maximum_filter(dist_transform, size=20) == dist_transform
markers = ndimage.label(local_max)[0]

# Watershed segmentation
