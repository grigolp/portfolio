The Challenge

Mobile traffic puzzle games require designers to create solvable levels with specific difficulty ratings. Manual testing was taking 1-2 hours per complex level.

Problem Complexity

  • State space explosion: Traditional BFS approach O(V!) for V vehicles
  • Real-time requirement: Designers need instant feedback during level creation
  • Edge cases: Must detect impossible configurations early
  • No ML data: Limited training examples, rule-based approach required

Novel Algorithm Approach

Iterative Removal Strategy

Instead of searching all possible moves, identify and remove vehicles that can exit in current state:

# Novel O(V²) iterative removal algorithm def validate_puzzle(board, vehicles): removable = True while removable: removable = False for vehicle in vehicles: if can_exit(vehicle, board): remove_vehicle(vehicle, board) vehicles.remove(vehicle) removable = True break return len(vehicles) == 0 # All vehicles removed = solvable # Complexity: O(V²) iterations vs O(V!) state exploration
